# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in WorkBee, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please report vulnerabilities via [GitHub Security Advisories](https://github.com/willfarrell/workbee/security/advisories/new).

## Scope

This policy applies to:

- `@work-bee/core`
- `@work-bee/cache-control`
- `@work-bee/fallback`
- `@work-bee/inactivity`
- `@work-bee/logger`
- `@work-bee/offline`
- `@work-bee/save-data`
- `@work-bee/session`

## Response

We aim to acknowledge reports within 48 hours and provide a fix within 7 days for confirmed vulnerabilities.
